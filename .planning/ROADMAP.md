### Phase 10: Level System
**Goal**: Users progress through named tiers with clear visual identity
**Depends on**: Phase 9
**Requirements**: LEVL-01, LEVL-02, LEVL-03
**Success Criteria** (what must be TRUE):
  1. App displays 5-6 named progression tiers (e.g., Novice -> Master)
  2. Each tier has a distinct color visible in UI
  3. User can see their current tier name and progress percentage toward next tier
  4. Tier thresholds are based on games played or mastery metrics

### Phase 12: Onboarding
**Goal**: New users understand triads and scoring before first quiz
**Depends on**: Phase 11
**Requirements**: ONBD-01, ONBD-02, ONBD-03
**Success Criteria** (what must be TRUE):
  1. User with gamesPlayed = 0 sees onboarding on app launch
  2. User with gamesPlayed > 0 never sees onboarding
  3. Onboarding has 2-3 screens explaining what triads are and how scoring works
  4. User can skip onboarding at any point via visible skip button
