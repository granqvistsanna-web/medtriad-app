## Objective

Document the design system so future development maintains consistency. The color palette, spacing grid, typography hierarchy, and motion presets should be clearly documented in a single reference file.

**Design Principle:** A design system is only useful if it's documented. Future you (and others) need a quick reference.

## Context

The design system exists in theme.ts but lacks documentation:
- Colors are semantic but benefits from visual reference
- Spacing is 8pt grid but not explicitly documented
- Typography hierarchy is well-structured
- Easings are new and need explanation

Creating a DESIGN-SYSTEM.md provides:
1. Quick reference for all tokens
2. Usage examples
3. Motion principles

## Motion Principles

1. **Springs over timing** — Use withSpring for position/scale changes
2. **Easing on everything** — No linear motion
3. **Stagger grouped elements** — Build anticipation with delays
4. **Soft, weighted, physical** — Motion should feel real
5. **Nothing snaps** — Everything settles

### Entry Animation Pattern

\`\`\`typescript
entering={FadeInUp.delay(N * Durations.stagger).duration(Durations.normal).springify()}
\`\`\`

### Press Animation Pattern

\`\`\`typescript
const handlePressIn = () => {
  scale.value = withSpring(0.98, Easings.press);
};

const handlePressOut = () => {
  scale.value = withSpring(1, Easings.press);
};
\`\`\`

### Pop Effect Pattern

\`\`\`typescript
scale.value = withSequence(
  withSpring(1.35, Easings.pop),   // Overshoot
  withSpring(1, Easings.gentle)    // Settle
);
\`\`\`
